Overview

ReqRight is a browser-based systems engineering workbench that helps you write, analyse, and manage requirements using the INCOSE Guide to Writing Requirements (GtWR) version 4 standard. It includes six integrated modules covering the full requirements lifecycle.

Quick Start

ReqRight requires no installation. Open reqright.html in any modern browser (Chrome, Firefox, Edge, Safari) and you're ready to go.

First Steps

  1. Create a requirement ‚Äî Click + New or press Ctrl+N
  2. Write your statement ‚Äî Type in the editor, using "shall" as the obligation keyword
  3. Check the quality score ‚Äî The right panel shows your INCOSE score in real-time
  4. Fix violations ‚Äî Click ‚ö° AI Improve to auto-fix common issues
  5. Save your work ‚Äî Press Ctrl+S to export as JSON (auto-save is always active)
üí° Load the included example project (AegisGuard Smart Home Security System) to explore all features with realistic data.

Quality Analysis

Every requirement is scored in real-time against all 42 INCOSE GtWR v4 rules. The quality panel shows:

  • Score ‚Äî 0‚Äì100% based on weighted pass/fail results
  • Grade ‚Äî A+ (95+), A (90+), B+ (85+), B (80+), C+ (75+), C (70+), D (60+), F (<60)
  • Rule-by-rule breakdown ‚Äî Each rule shows pass (‚úì), fail (‚úï), or review (‚ö†)

Rules are grouped into 14 categories: Accuracy, Concision, Non-Ambiguity, Singularity, Completeness, Realism, Conditions, Uniqueness, Abstraction, Quantifiers, Tolerance, Quantification, Uniformity, and Modularity.

AI Improvement

Click ‚ö° AI Improve to automatically rewrite a requirement to fix common quality violations. The rule-based engine handles:

  • Passive voice ‚Üí Active voice
  • Vague terms ("adequate", "fast", "reliable") ‚Üí Specific placeholders
  • Escape clauses ("where possible", "as needed") ‚Üí Removal
  • Superfluous infinitives ("shall be able to") ‚Üí Simplified
  • Absolutes ("always", "never") ‚Üí Achievable targets
  • Abbreviations ‚Üí Full words
  • Missing "shall" keyword ‚Üí Inserted
  • Missing period ‚Üí Added
‚ö† The AI engine is rule-based and runs entirely offline ‚Äî no LLM or API key required. Always review AI-improved text to ensure it captures your intent.

Version History

ReqRight automatically tracks all changes to requirement fields. The Version History panel shows a timeline (newest first) with:

  • Field name ‚Äî Which field changed (Statement, Type, Level, Priority, Status, etc.)
  • Author ‚Äî Who made the change (your reviewer name, or "AI Engine" for AI rewrites)
  • Timestamp ‚Äî When the change occurred
  • Diff ‚Äî old value ‚Üí new value

Text/statement changes are tracked with a 1-second debounce to avoid logging every keystroke. AI rewrites are logged with before/after snippets and colour-coded in purple.

Auto-Save & Storage

ReqRight automatically saves your project to the browser's localStorage on every change. This means your data persists between sessions ‚Äî even if you close the browser tab. However, clearing browser data will erase your project.

‚ö† Always export to JSON regularly as a backup. LocalStorage has size limits (~5-10MB depending on browser) and can be cleared by the browser.

FAQ

Where is my data stored?

All data is stored in your browser's localStorage. Nothing is sent to any server. Export to JSON for backup.

Can multiple people collaborate?

ReqRight is a single-user tool. For collaboration, export and share JSON files. The Comments & Review system supports different reviewer names but works within one browser session.

What happens if I clear my browser data?

Your project will be lost. Always maintain JSON backups via Ctrl+S or File ‚Üí Save.

Is the AI engine sending data to the cloud?

No. The AI Improve feature is entirely rule-based and runs offline in JavaScript. No LLM or external API is used.

Can I import from IBM DOORS?

Export your DOORS data as CSV or ReqIF, then import into ReqRight. Column mapping handles different header formats.

What browsers are supported?

Chrome, Firefox, Edge, and Safari (latest versions). ReqRight uses modern JavaScript features and may not work in older browsers.

How do I report a bug?

Open an issue on GitHub with steps to reproduce.
